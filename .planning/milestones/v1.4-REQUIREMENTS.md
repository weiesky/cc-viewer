# M1.4: 细节修复与上下文结构化 — 需求文档

**版本**: v1.4
**创建时间**: 2026-04-24

---

## 背景

M1.3 完成了可视化编辑器布局重构与 Sketch MCP 集成。M1.4 聚焦两个遗留问题：

1. **用户体验 bug**：切换模式后 iframe URL 重置，每次都要重新输入项目地址，体验差
2. **架构债务**：`buildElementContext()` 将指令与数据混合在硬编码中文字符串中，难以维护和扩展

---

## 需求

### R1: iframe URL 状态持久化

- **现象**：用户在 visual 模式输入 `http://localhost:3000`，切换到其他视图后再切回，URL 被清空
- **期望**：`previewUrl` 在整个应用生命周期内持久化，切换模式不丢失
- **验收**：手动切换模式 3 次，URL 保持不变

### R2: XML 结构化元素上下文

- **现象**：`buildElementContext()` 输出 `"请修改以下 React 组件中的元素：\n组件：Button\n..."`，指令与数据强耦合
- **期望**：输出 XML 格式，纯数据描述，用户的自然语言 prompt 另行附加
  ```xml
  <selected-element>
    <tag>button</tag>
    <class>btn-primary</class>
    <id></id>
    <screenshot>/tmp/element-xxx.png</screenshot>
    <component>Button</component>
    <file>src/components/Button.jsx</file>
    <selector>button.btn-primary</selector>
    <text>提交</text>
    <styles>
      <color>rgb(255,255,255)</color>
      <background-color>rgb(24,144,255)</background-color>
      <font-size>14px</font-size>
    </styles>
  </selected-element>
  ```
- **验收**：选中元素后按 Enter，终端发送内容格式为 `<selected-element>...</selected-element>\n{用户输入}`
