# getDiagnostics (mcp__ide__getDiagnostics)

## คำจำกัดความ

รับข้อมูลการวินิจฉัยภาษาจาก VS Code รวมถึงข้อผิดพลาดทางไวยากรณ์ ข้อผิดพลาดประเภท คำเตือน lint ฯลฯ

## พารามิเตอร์

| พารามิเตอร์ | ประเภท | จำเป็น | คำอธิบาย |
|------|------|------|------|
| `uri` | string | ไม่ | URI ของไฟล์ หากไม่ระบุจะรับข้อมูลวินิจฉัยของไฟล์ทั้งหมด |

## สถานการณ์การใช้งาน

**เหมาะสำหรับ:**
- ตรวจสอบปัญหาเชิงความหมายของโค้ด เช่น ไวยากรณ์ ประเภท lint
- ตรวจสอบว่าการแก้ไขโค้ดทำให้เกิดข้อผิดพลาดใหม่หรือไม่
- ใช้แทนคำสั่ง Bash เพื่อตรวจสอบคุณภาพโค้ด

**ไม่เหมาะสำหรับ:**
- รันการทดสอบ — ควรใช้ Bash
- ตรวจสอบข้อผิดพลาดขณะรัน — ควรใช้ Bash เพื่อรันโค้ด

## ข้อควรระวัง

- นี่คือเครื่องมือ MCP (Model Context Protocol) ที่จัดเตรียมโดยการรวมกับ IDE
- ใช้ได้เฉพาะในสภาพแวดล้อม VS Code / IDE
- ควรใช้เครื่องมือนี้แทนคำสั่ง Bash เพื่อตรวจสอบปัญหาโค้ด

## ความสำคัญใน cc-viewer

getDiagnostics เป็นเครื่องมือ MCP ปรากฏในอาร์เรย์ `tools` ของบันทึกคำร้องขอด้วยชื่อ `mcp__ide__getDiagnostics` การเรียกและส่งคืนเป็นไปตามรูปแบบ `tool_use` / `tool_result` มาตรฐาน การเพิ่ม/ลบเครื่องมือ MCP ทำให้อาร์เรย์ tools เปลี่ยนแปลง ซึ่งอาจทำให้เกิดการสร้างแคชใหม่
