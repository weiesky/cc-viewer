# getDiagnostics (mcp__ide__getDiagnostics)

## Визначення

Отримує діагностичну інформацію мови з VS Code, включаючи синтаксичні помилки, помилки типів, попередження lint тощо.

## Параметри

| Параметр | Тип | Обов'язковий | Опис |
|----------|-----|--------------|------|
| `uri` | string | Ні | URI файлу. Якщо не вказано, отримує діагностику для всіх файлів |

## Сценарії використання

**Підходить для:**
- Перевірка синтаксичних, типових, lint та інших семантичних проблем коду
- Перевірка після редагування коду, чи не з'явилися нові помилки
- Заміна команд Bash для перевірки якості коду

**Не підходить для:**
- Запуск тестів — слід використовувати Bash
- Перевірка помилок виконання — слід використовувати Bash для виконання коду

## Примітки

- Це MCP (Model Context Protocol) інструмент, наданий інтеграцією з IDE
- Доступний лише в середовищі VS Code / IDE
- Надавайте перевагу цьому інструменту замість команд Bash для перевірки проблем коду

## Значення в cc-viewer

getDiagnostics — це MCP-інструмент, який з'являється в масиві `tools` журналу запитів під назвою `mcp__ide__getDiagnostics`. Його виклики та повернення відповідають стандартному шаблону `tool_use` / `tool_result`. Додавання або видалення MCP-інструментів призводить до зміни масиву tools, що може спричинити перебудову кешу.
