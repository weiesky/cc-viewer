# CC-Viewer

Система моніторингу запитів Claude Code, яка захоплює та візуалізує всі API-запити та відповіді в реальному часі (оригінальний текст, без цензури). Допомагає розробникам відстежувати свій Context, щоб переглядати та усувати проблеми під час Vibe Coding.
Остання версія CC-Viewer також пропонує рішення для серверного веб-програмування, а також інструменти для мобільного програмування. Ласкаво просимо використовувати у своїх проєктах. У майбутньому будуть додані нові функції плагінів та підтримка хмарного розгортання.

## Використання

### Встановлення

```bash
npm install -g cc-viewer --registry=https://registry.npmjs.org
```

### Режим програмування

ccv є прямою заміною claude. Усі параметри передаються до claude, одночасно запускається Web Viewer.

```bash
ccv                    # == claude (інтерактивний режим)
ccv -c                 # == claude --continue (продовжити останню розмову)
ccv -r                 # == claude --resume (відновити розмову)
ccv -p "hello"         # == claude --print "hello" (режим друку)
ccv --d                # == claude --dangerously-skip-permissions (ярлик)
ccv --model opus       # == claude --model opus
```

Команда, яку автор використовує найчастіше:
```
ccv -c --d             # == claude --continue --dangerously-skip-permissions
```

Після запуску режиму програмування веб-сторінка відкриється автоматично.

Ви можете використовувати claude прямо на веб-сторінці, одночасно переглядаючи повні повідомлення запитів та зміни коду.

І що ще краще — ви навіть можете програмувати з мобільного!


### Режим журналювання

⚠️ Якщо ви все ще віддаєте перевагу рідному інструменту claude або плагіну VS Code, використовуйте цей режим.

У цьому режимі при запуску ```claude``` або ```claude --dangerously-skip-permissions```

автоматично запускається процес журналювання, який записує логи запитів до ~/.claude/cc-viewer/*ваш_проєкт*/дата.jsonl

Запуск режиму журналювання:
```bash
ccv -logger
```

Коли конкретний порт не може бути виведений у консолі, порт за замовчуванням для першого екземпляра — 127.0.0.1:7008. Для кількох одночасних екземплярів порти збільшуються послідовно: 7009, 7010.

Ця команда автоматично визначає спосіб встановлення Claude Code (NPM або Native Install) і адаптується відповідно.

- **NPM-версія claude code**: Автоматично впроваджує скрипт перехоплення в `cli.js` Claude Code.
- **Native-версія claude code**: Автоматично виявляє бінарний файл `claude`, налаштовує локальний прозорий проксі та встановлює Zsh Shell Hook для автоматичного перенаправлення трафіку.
- Цей проєкт рекомендує використовувати Claude Code, встановлений через NPM.

Видалення режиму журналювання:
```bash
ccv --uninstall
```

### Усунення неполадок (Troubleshooting)

Якщо у вас виникли проблеми з запуском, є остаточне рішення:
Крок 1: Відкрийте Claude Code в будь-якому каталозі;
Крок 2: Дайте Claude Code таку інструкцію:
```
Я встановив npm-пакет cc-viewer, але після виконання ccv він все ще не працює належним чином. Перевір cli.js та findcc.js у cc-viewer і адаптуй локальне розгортання Claude Code відповідно до конкретного середовища. Обмеж зміни findcc.js наскільки це можливо.
```
Дозволити Claude Code самому перевірити помилки — ефективніше, ніж запитувати будь-кого або читати будь-яку документацію!

Після виконання вищезазначених інструкцій findcc.js буде оновлено. Якщо ваш проєкт часто потребує локального розгортання, або форкнутий код часто потребує вирішення проблем з встановленням, просто збережіть цей файл і скопіюйте його наступного разу. Наразі багато проєктів та компаній розгортають Claude Code не на Mac, а на серверних хостингах. Тому автор виокремив findcc.js для зручності відстеження оновлень вихідного коду cc-viewer.

### Інші допоміжні команди

Довідка:
```bash
ccv -h
```

### Перевизначення конфігурації (Configuration Override)

Якщо вам потрібно використовувати користувацьку кінцеву точку API (наприклад, корпоративний проксі), просто налаштуйте її в `~/.claude/settings.json` або встановіть змінну середовища `ANTHROPIC_BASE_URL`. `ccv` автоматично виявить і правильно перенаправить запити.

### Тихий режим (Silent Mode)

За замовчуванням `ccv` працює в тихому режимі при обгортанні `claude`, забезпечуючи чистий вивід терміналу, ідентичний рідному досвіду. Усі логи захоплюються у фоновому режимі та доступні за адресою `http://localhost:7008`.

Після завершення налаштування використовуйте команду `claude` як зазвичай. Відвідайте `http://localhost:7008`, щоб переглянути інтерфейс моніторингу.


## Клієнтська версія

CC-Viewer доступний як клієнтський додаток, який можна завантажити на GitHub.
[Завантажити тут](https://github.com/weiesky/cc-viewer/releases)
Наразі клієнтська версія перебуває на етапі тестування — якщо виникнуть проблеми, ви можете будь-коли надіслати зворотний зв'язок. Крім того, для використання cc-viewer необхідно, щоб на вашому комп'ютері був встановлений Claude Code.
Важливо розуміти: cc-viewer — це лише «одяг» для працівника (Claude Code). Без Claude Code одяг не може працювати самостійно.

### Режим журналювання (перегляд повних розмов Claude Code)

<img width="1500" height="768" alt="image" src="https://github.com/user-attachments/assets/a8a9f3f7-d876-4f6b-a64d-f323a05c4d21" />


- Захоплює всі API-запити від Claude Code в реальному часі, гарантуючи оригінальний текст, а не цензуровані логи (це дуже важливо!!!)
- Автоматично ідентифікує та маркує запити Main Agent і Sub Agent (підтипи: Plan, Search, Bash)
- Запити MainAgent підтримують Body Diff JSON, показуючи відмінності від попереднього запиту MainAgent у згорнутому вигляді (відображаються лише змінені/нові поля)
- Кожен запит показує вбудовану статистику використання Token (вхідні/вихідні Token, створення/читання кешу, коефіцієнт влучень)
- Сумісний з Claude Code Router (CCR) та іншими проксі-сценаріями — зіставлення запитів через шаблон API-шляху як резервний варіант

### Режим розмови

Натисніть кнопку "Режим розмови" у верхньому правому куті, щоб розібрати повну історію розмов Main Agent як чат-інтерфейс:

<img width="1500" height="764" alt="image" src="https://github.com/user-attachments/assets/725b57c8-6128-4225-b157-7dba2738b1c6" />


- Відображення Agent Team поки не підтримується
- Повідомлення користувача вирівняні праворуч (синя бульбашка), відповіді Main Agent — ліворуч (темна бульбашка)
- Блоки `thinking` за замовчуванням згорнуті, рендеряться як Markdown. Натисніть, щоб розгорнути та побачити процес мислення; підтримується переклад одним кліком (функція поки нестабільна)
- Повідомлення вибору користувача (AskUserQuestion) відображаються у форматі запитання-відповідь
- Двостороння синхронізація режимів: при переключенні в режим розмови автоматично переходить до розмови обраного запиту; при поверненні в оригінальний режим автоматично переходить до обраного запиту
- Панель налаштувань: можна перемикати стан згортання за замовчуванням для результатів інструментів та блоків thinking
- Мобільний перегляд розмов: у мобільному режимі CLI натисніть кнопку "Перегляд розмов" у верхній панелі, щоб відкрити перегляд тільки для читання та переглядати повну історію розмов на мобільному

### Автоматичні оновлення

CC-Viewer автоматично перевіряє оновлення при запуску (максимум раз на 4 години). В межах однієї мажорної версії (наприклад, 1.x.x -> 1.y.z) оновлюється автоматично, зміни набирають чинності при наступному запуску. При зміні мажорної версії лише показується сповіщення.

Автоматичне оновлення слідує глобальній конфігурації Claude Code `~/.claude/settings.json`. Якщо Claude Code вимкнув автоматичне оновлення (`autoUpdates: false`), CC-Viewer також пропускає автоматичне оновлення.
