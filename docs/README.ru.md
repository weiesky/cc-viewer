# CC-Viewer

Система мониторинга запросов Claude Code — перехватывает и визуализирует в реальном времени все API-запросы и ответы Claude Code (исходный текст, без сокращений). Позволяет разработчикам отслеживать свой Context для удобного анализа и отладки в процессе Vibe Coding.
Последняя версия CC-Viewer также предоставляет решения для серверного веб-развёртывания и инструменты для мобильной разработки. Приглашаем использовать в своих проектах — в будущем будет доступно больше плагинов и поддержка облачного развёртывания.

## Использование

### Установка

```bash
npm install -g cc-viewer --registry=https://registry.npmjs.org
```

### Режим программирования

ccv — это прямая замена claude, все параметры передаются в claude, одновременно запускается Web Viewer.

```bash
ccv                    # == claude (интерактивный режим)
ccv -c                 # == claude --continue (продолжить предыдущий диалог)
ccv -r                 # == claude --resume (возобновить диалог)
ccv -p "hello"         # == claude --print "hello" (режим печати)
ccv --d                # == claude --dangerously-skip-permissions (быстрый доступ)
ccv --model opus       # == claude --model opus
```

Наиболее часто используемая автором команда
```
ccv -c --d             # == claude --continue --dangerously-skip-permissions
```

После запуска режима программирования автоматически откроется веб-страница.

Вы можете напрямую использовать claude на веб-странице, а также просматривать полные тела запросов и изменения кода.

И что ещё более привлекательно — вы даже можете программировать с мобильного устройства!


### Режим логирования

⚠️ Если вы по-прежнему предпочитаете использовать нативный инструмент claude или плагин VS Code, используйте этот режим.

В этом режиме при запуске ```claude``` или ```claude --dangerously-skip-permissions```

автоматически запускается процесс логирования, который записывает журналы запросов в ~/.claude/cc-viewer/*вашпроект*/date.jsonl

Запуск режима логирования:
```bash
ccv -logger
```

Когда в консоли невозможно вывести конкретный порт, первый порт запуска по умолчанию — 127.0.0.1:7008. При наличии нескольких экземпляров номера портов увеличиваются последовательно, например 7009, 7010.

Эта команда автоматически определяет способ установки локального Claude Code (NPM или Native Install) и адаптируется к нему.

- **NPM версия claude code**: автоматически внедряет перехватывающий скрипт в `cli.js` Claude Code.
- **Native версия claude code**: автоматически обнаруживает бинарный файл `claude`, настраивает локальный прозрачный прокси и устанавливает Zsh Shell Hook для автоматической переадресации трафика.
- Рекомендуется использовать claude code, установленный через npm.

Удаление режима логирования:
```bash
ccv --uninstall
```

### Устранение неполадок (Troubleshooting)

Если у вас возникли проблемы с запуском, есть универсальный способ диагностики:
Шаг первый: откройте claude code в любой директории;
Шаг второй: дайте claude code следующую инструкцию:
```
Я установил npm-пакет cc-viewer, но после выполнения ccv он не работает корректно. Посмотри cli.js и findcc.js пакета cc-viewer, определи особенности текущего окружения и адаптируй под локальный способ развёртывания Claude Code. При адаптации старайся ограничить изменения файлом findcc.js.
```
Позволить Claude Code самому проверить ошибки — это эффективнее, чем консультироваться с кем-либо или читать документацию!

После выполнения инструкции файл findcc.js будет обновлён. Если ваш проект часто требует локального развёртывания или вы работаете с форком и регулярно решаете проблемы установки — сохраните этот файл, и в следующий раз просто скопируйте его. На данный момент многие проекты и компании используют claude code не на Mac, а на серверном хостинге, поэтому автор вынес findcc.js в отдельный файл для удобства отслеживания обновлений исходного кода cc-viewer.

### Другие вспомогательные команды

Справка:
```bash
ccv -h
```

### Переопределение конфигурации (Configuration Override)

Если вам нужно использовать пользовательскую конечную точку API (например, корпоративный прокси), просто настройте `~/.claude/settings.json` или задайте переменную окружения `ANTHROPIC_BASE_URL`. `ccv` автоматически распознает и корректно перенаправит запросы.

### Тихий режим (Silent Mode)

По умолчанию `ccv` работает в тихом режиме при обёртывании `claude`, обеспечивая чистый вывод в терминале, идентичный нативному. Все логи перехватываются в фоне и доступны по адресу `http://localhost:7008`.

После настройки просто используйте команду `claude` как обычно. Откройте `http://localhost:7008` для просмотра интерфейса мониторинга.


## Клиентская версия

CC-Viewer доступен в виде клиентского приложения, которое можно скачать на GitHub.
[Скачать здесь](https://github.com/weiesky/cc-viewer/releases)
На данный момент клиентская версия находится на этапе тестирования — если возникнут проблемы, вы можете в любое время оставить обратную связь. Кроме того, для использования cc-viewer необходимо, чтобы на вашем компьютере был установлен Claude Code.
Важно понимать: cc-viewer — это всего лишь «одежда» для работника (Claude Code). Без Claude Code одежда не может работать самостоятельно.

### Режим логирования (просмотр полных сессий claude code)

**Сжатие логов**
По поводу логов автор хочет заявить, что официальные определения Anthropic не были изменены, чтобы гарантировать целостность логов.
Однако, поскольку отдельные записи логов 1M opus на поздних этапах становятся чрезвычайно объёмными, благодаря оптимизации логов MainAgent, выполненной автором, размер удалось уменьшить как минимум на 66% без использования gzip.
Метод парсинга этих сжатых логов можно извлечь из текущего репозитория.

### Автоматическое обновление

CC-Viewer автоматически проверяет обновления при запуске (не чаще одного раза в 4 часа). В пределах одной мажорной версии (например, 1.x.x → 1.y.z) обновления применяются автоматически и вступают в силу при следующем запуске. При смене мажорной версии отображается только уведомление.

Автоматическое обновление следует глобальной конфигурации Claude Code в `~/.claude/settings.json`. Если в Claude Code автоматические обновления отключены (`autoUpdates: false`), CC-Viewer также пропустит автоматическое обновление.
