# CC-Viewer

ระบบตรวจสอบคำขอ Claude Code ที่จับภาพและแสดงผลคำขอและการตอบกลับ API ทั้งหมดแบบเรียลไทม์ (ข้อความต้นฉบับ ไม่ถูกตัดทอน) ช่วยให้นักพัฒนาสามารถตรวจสอบ Context ของตัวเองได้ง่ายขึ้น เพื่อทบทวนและแก้ไขปัญหาระหว่างการทำ Vibe Coding
เวอร์ชันล่าสุดของ CC-Viewer ยังมีโซลูชันสำหรับการเขียนโปรแกรมเว็บบนเซิร์ฟเวอร์ รวมถึงเครื่องมือสำหรับการเขียนโปรแกรมบนมือถือ ยินดีต้อนรับทุกคนนำไปใช้ในโปรเจกต์ของตัวเอง ในอนาคตจะเปิดฟีเจอร์ปลั๊กอินเพิ่มเติมและรองรับการ deploy บนคลาวด์

## วิธีใช้งาน

### การติดตั้ง

```bash
npm install -g cc-viewer --registry=https://registry.npmjs.org
```

### โหมดเขียนโปรแกรม

ccv เป็นตัวแทนโดยตรงของ claude พารามิเตอร์ทั้งหมดจะถูกส่งต่อไปยัง claude พร้อมกับเปิด Web Viewer ขึ้นมาด้วย

```bash
ccv                    # == claude (โหมดโต้ตอบ)
ccv -c                 # == claude --continue (สานต่อบทสนทนาล่าสุด)
ccv -r                 # == claude --resume (กลับมาบทสนทนาเดิม)
ccv -p "hello"         # == claude --print "hello" (โหมดพิมพ์)
ccv --d                # == claude --dangerously-skip-permissions (ทางลัด)
ccv --model opus       # == claude --model opus
```

คำสั่งที่ผู้เขียนใช้บ่อยที่สุดคือ:
```
ccv -c --d             # == claude --continue --dangerously-skip-permissions
```

หลังจากเริ่มโหมดเขียนโปรแกรม หน้าเว็บจะเปิดขึ้นโดยอัตโนมัติ

คุณสามารถใช้ claude ได้โดยตรงบนหน้าเว็บ พร้อมดูข้อความคำขอทั้งหมดและตรวจสอบการเปลี่ยนแปลงโค้ดได้

และที่น่าตื่นเต้นกว่านั้น — คุณสามารถเขียนโปรแกรมจากมือถือได้เลย!


### โหมดบันทึก Log

⚠️ หากคุณยังคงต้องการใช้เครื่องมือ claude ดั้งเดิม หรือปลั๊กอิน VS Code ให้ใช้โหมดนี้

ในโหมดนี้ เมื่อเริ่ม ```claude``` หรือ ```claude --dangerously-skip-permissions```

จะมีกระบวนการบันทึก log เริ่มทำงานโดยอัตโนมัติ บันทึกคำขอไปที่ ~/.claude/cc-viewer/*โปรเจกต์ของคุณ*/วันที่.jsonl

เริ่มโหมดบันทึก log:
```bash
ccv -logger
```

เมื่อไม่สามารถแสดงพอร์ตในคอนโซลได้ พอร์ตเริ่มต้นของอินสแตนซ์แรกคือ 127.0.0.1:7008 หากมีหลายอินสแตนซ์พร้อมกัน จะใช้พอร์ตถัดไป เช่น 7009, 7010

คำสั่งนี้จะตรวจจับวิธีการติดตั้ง Claude Code (NPM หรือ Native Install) โดยอัตโนมัติและปรับตัวตาม

- **Claude code เวอร์ชัน NPM**: ฉีดสคริปต์ดักจับเข้าไปใน `cli.js` ของ Claude Code โดยอัตโนมัติ
- **Claude code เวอร์ชัน Native**: ตรวจจับไฟล์ไบนารี `claude` โดยอัตโนมัติ กำหนดค่า proxy ในเครื่อง และตั้งค่า Zsh Shell Hook สำหรับการส่งต่อทราฟฟิกอัตโนมัติ
- โปรเจกต์นี้แนะนำให้ใช้ Claude Code ที่ติดตั้งผ่าน NPM

ถอนการติดตั้งโหมดบันทึก log:
```bash
ccv --uninstall
```

### การแก้ไขปัญหา (Troubleshooting)

หากคุณพบปัญหาในการเริ่มต้น มีวิธีแก้ไขสุดท้าย:
ขั้นตอนที่ 1: เปิด Claude Code ในไดเรกทอรีใดก็ได้;
ขั้นตอนที่ 2: ให้คำสั่งต่อไปนี้แก่ Claude Code:
```
ฉันติดตั้งแพ็กเกจ npm cc-viewer แล้ว แต่หลังจากรัน ccv ยังคงทำงานไม่ถูกต้อง ช่วยตรวจสอบ cli.js และ findcc.js ของ cc-viewer แล้วปรับการ deploy Claude Code ในเครื่องตามสภาพแวดล้อมจริง โดยจำกัดการแก้ไขให้อยู่ใน findcc.js เท่าที่เป็นไปได้
```
ให้ Claude Code ตรวจสอบข้อผิดพลาดด้วยตัวเอง มีประสิทธิภาพมากกว่าการถามใครหรืออ่านเอกสารใดๆ!

หลังจากทำตามคำสั่งข้างต้นเสร็จ findcc.js จะถูกอัปเดต หากโปรเจกต์ของคุณต้อง deploy ในเครื่องบ่อยๆ หรือโค้ดที่ fork ไปต้องแก้ปัญหาการติดตั้งบ่อย ก็แค่เก็บไฟล์นี้ไว้แล้วคัดลอกโดยตรงในครั้งถัดไป ปัจจุบันหลายโปรเจกต์และบริษัทไม่ได้ deploy Claude Code บน Mac แต่ใช้การ deploy บนเซิร์ฟเวอร์ ดังนั้นผู้เขียนจึงแยก findcc.js ออกมา เพื่อให้ง่ายต่อการติดตามอัปเดตของซอร์สโค้ด cc-viewer

### คำสั่งช่วยเหลืออื่นๆ

ดูข้อมูลเพิ่มเติม:
```bash
ccv -h
```

### การแทนที่การตั้งค่า (Configuration Override)

หากคุณต้องการใช้ API endpoint ที่กำหนดเอง (เช่น proxy ขององค์กร) เพียงกำหนดค่าใน `~/.claude/settings.json` หรือตั้งค่าตัวแปรสภาพแวดล้อม `ANTHROPIC_BASE_URL` `ccv` จะตรวจจับและส่งต่อคำขอได้อย่างถูกต้องโดยอัตโนมัติ

### โหมดเงียบ (Silent Mode)

โดยค่าเริ่มต้น `ccv` จะทำงานในโหมดเงียบเมื่อครอบ `claude` เพื่อให้เอาต์พุตของเทอร์มินัลสะอาดและเหมือนกับประสบการณ์ดั้งเดิม log ทั้งหมดจะถูกจับภาพในเบื้องหลังและสามารถดูได้ที่ `http://localhost:7008`

หลังจากตั้งค่าเสร็จ ใช้คำสั่ง `claude` ตามปกติ เข้าถึง `http://localhost:7008` เพื่อดูอินเทอร์เฟซการตรวจสอบ


## เวอร์ชันไคลเอนต์

cc-viewer มีเวอร์ชันไคลเอนต์เดสก์ท็อป คุณสามารถดาวน์โหลดได้จาก GitHub
[ลิงก์ดาวน์โหลด](https://github.com/weiesky/cc-viewer/releases)
เวอร์ชันไคลเอนต์อยู่ในขั้นตอนการทดสอบ หากพบปัญหาสามารถแจ้งข้อเสนอแนะได้ตลอดเวลา นอกจากนี้ ข้อกำหนดเบื้องต้นของการใช้ cc-viewer คือคุณต้องมี Claude Code ติดตั้งอยู่ในเครื่อง
สิ่งสำคัญที่ต้องทราบคือ: cc-viewer เป็นเพียง "เสื้อผ้า" ของคนงาน (Claude Code) เท่านั้น — หากไม่มี Claude Code เสื้อผ้าก็ไม่สามารถทำงานได้ด้วยตัวเอง

### โหมดบทสนทนา

คลิกปุ่ม "โหมดบทสนทนา" ที่มุมขวาบน เพื่อแปลงประวัติบทสนทนาทั้งหมดของ Main Agent เป็นอินเทอร์เฟซแชท:

<img width="1500" height="764" alt="image" src="https://github.com/user-attachments/assets/725b57c8-6128-4225-b157-7dba2738b1c6" />


- ยังไม่รองรับการแสดงผล Agent Team
- ข้อความผู้ใช้จัดชิดขวา (ฟองสีน้ำเงิน), คำตอบ Main Agent จัดชิดซ้าย (ฟองสีเข้ม)
- บล็อก `thinking` ยุบโดยค่าเริ่มต้น แสดงผลเป็น Markdown คลิกเพื่อดูกระบวนการคิด; รองรับการแปลด้วยคลิกเดียว (ฟีเจอร์ยังไม่เสถียร)
- ข้อความเลือกของผู้ใช้ (AskUserQuestion) แสดงในรูปแบบถาม-ตอบ
- การซิงค์โหมดสองทาง: เมื่อสลับไปโหมดบทสนทนา จะนำทางไปยังบทสนทนาของคำขอที่เลือกโดยอัตโนมัติ; เมื่อสลับกลับไปโหมดต้นฉบับ จะนำทางไปยังคำขอที่เลือกโดยอัตโนมัติ
- แผงการตั้งค่า: สามารถสลับสถานะการยุบเริ่มต้นของผลลัพธ์เครื่องมือและบล็อก thinking
- มุมมองบทสนทนาบนมือถือ: ในโหมด CLI บนมือถือ คลิกปุ่ม "ดูบทสนทนา" ที่แถบด้านบน เพื่อเปิดมุมมองบทสนทนาแบบอ่านอย่างเดียว เพื่อเรียกดูประวัติบทสนทนาทั้งหมดบนมือถือ

### อัปเดตอัตโนมัติ

CC-Viewer ตรวจสอบอัปเดตโดยอัตโนมัติเมื่อเริ่มต้น (สูงสุดครั้งเดียวทุก 4 ชั่วโมง) ภายในเวอร์ชันหลักเดียวกัน (เช่น 1.x.x -> 1.y.z) จะอัปเดตอัตโนมัติ มีผลในการเริ่มต้นครั้งถัดไป เมื่อเปลี่ยนเวอร์ชันหลัก จะแสดงเฉพาะการแจ้งเตือน

การอัปเดตอัตโนมัติตามการตั้งค่าส่วนกลางของ Claude Code `~/.claude/settings.json` หาก Claude Code ปิดการอัปเดตอัตโนมัติ (`autoUpdates: false`) CC-Viewer จะข้ามการอัปเดตอัตโนมัติเช่นกัน

### รองรับหลายภาษา
